## Repository Layout

This is a personal dotfiles repo using a **Stow-compatible directory structure**: each top-level package folder mirrors the target path from `$HOME`. For example, `hypr/.config/hypr/` symlinks to `~/.config/hypr/`. Packages: `btop`, `fish`, `fontconfig`, `ghostty`, `hypr`, `noctalia`, `nvim`, `starship`, `zsh`.

To deploy a package: `stow -d /home/naeem/dotfiles -t $HOME <package>`

Shared config lives on `main`. Machine-specific overrides are maintained as separate branches:

## Hyprland Config Architecture (`hypr/`)

`hyprland.conf` is a pure entry point — it only `source =`s other files in order:

```
env.conf       → Wayland/GPU/cursor env vars (must load first)
monitor.conf   → Display layout (monitorv2 blocks, machine-specific)
general.conf   → Gaps, borders, focus behavior
decoration.conf → Blur, rounding, shadows
animations.conf → Animation curves and speeds
input.conf     → Keyboard, mouse, touchpad
layouts.conf   → Master/dune layout settings
keybinds.conf  → All bind = directives
windowrules.conf → Per-app float/size/opacity rules
workspaces.conf  → Named workspaces, monitor assignments (machine-specific)
autostart.conf   → exec-once services and app launches
```

The final `source =` pulls in `noctalia-colors.conf` from `~/.config/hypr/noctalia/` (generated by Noctalia's colorscheme engine — not tracked directly in this repo).

### Useful Hyprland Commands

```bash
hyprctl reload          # Hot-reload all config files (no restart)
hyprctl monitors        # List connected outputs and their names
hyprctl clients         # List open windows with class/title (useful for windowrules)
hyprctl activewindow    # Show focused window details
```

## Noctalia Shell (`noctalia/`)

Noctalia is a Qt/QML desktop shell built on [Quickshell](https://quickshell.outfoxxed.me). Launched via:

```bash
qs -c noctalia-shell --no-duplicate
```

IPC calls from keybinds use: `qs -c noctalia-shell ipc call <action>`

**Plugin structure**: each plugin under `plugins/` contains:
- `manifest.json` — id, version, entryPoints, defaultSettings
- `BarWidget.qml` — bar icon/widget
- `Panel.qml` — expanded panel UI
- `Settings.qml` — settings pane
- `Main.qml` — service/logic entry

Colorschemes live in `colorschemes/` and are selected via `settings.json`. The active colorscheme generates `~/.config/hypr/noctalia/noctalia-colors.conf` which Hyprland sources.

## Neovim Config (`nvim/`)

LazyVim-based. Structure: `lua/config/` (autocmds, keymaps, lazy, options, colorscheme) and `lua/plugins/` for overrides. Plugin lockfile: `lazy-lock.json`.

## Zsh Config (`zsh/`)

Oh-My-Zsh with Powerlevel10k theme. Key files:
- `.zshrc` — plugin list, exports, aliases
- `.p10k.zsh` — prompt config (shared across machines)
- `.kube/kubie.yaml` — kubie context switcher config

`.kube/config*` and `.kube/configs/` are gitignored (kubeconfigs stay local).
